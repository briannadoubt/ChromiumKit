# Roadmap

ChromiumKit is currently pre-1.0. The immediate priority is making the package
safe and boring to adopt in real macOS app projects.

## Near Term

- stabilize the managed Xcode integration flow
- keep the pinned CEF release current on stable updates
- harden signing, notarization, and release diagnostics
- improve public docs, troubleshooting, and sample coverage

## After v0.1

- downloads API
- popup and new-window handling
- opt-in DevTools support
- better local file / custom scheme ergonomics
- richer SwiftUI examples and testing fixtures

## Longer Term

- App Sandbox investigation
- Mac App Store viability research
- more WebKit-style convenience APIs where the underlying behavior maps cleanly
- broader CI and launched-app smoke coverage
